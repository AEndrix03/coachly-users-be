# Task Completion Checklist
- Read the relevant files under `docs/` before implementing changes.
- Use Serena MCP tools first for discovery/navigation.
- Update or add tests for every behavior change.
- Run at least `.\\mvnw.cmd test` after meaningful code changes; use `.\\mvnw.cmd clean verify` when broader validation is needed.
- Update documentation when architecture, contracts, mappings, or workflows change.
- Create a scoped Conventional Commit at the end of each requested implementation step unless the user explicitly says not to.
- Leave the working tree clean after committing.
